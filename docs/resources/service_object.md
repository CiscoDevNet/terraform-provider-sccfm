# sccfm_service_object (Resource)

Provides a Service Object resource. Use this resource to create, read, update, and delete service objects in the Security Cloud Control unified object store.

## Example Usage

```terraform
resource "sccfm_service_object" "http" {
  name        = "HTTP"
  description = "HTTP traffic"
  protocol    = "TCP"
  value       = "80"
}

resource "sccfm_service_object" "high_ports" {
  name        = "high-ports"
  description = "High port range"
  protocol    = "TCP"
  value       = "8000-9000"
}

resource "sccfm_service_object" "with_overrides" {
  name        = "app-service"
  description = "Application service with per-target overrides"
  protocol    = "TCP"
  value       = "8080"

  overrides = [
    {
      target_id = "6dae078f-65a8-4cfc-8af8-9554976b5aae"
      protocol  = "TCP"
      value     = "9090"
    }
  ]
}
```

## Schema

### Required

- `name` (String) The name of the object.
- `protocol` (String) The protocol of the service object (e.g., `TCP`, `UDP`, `ICMP`).

### Optional

- `description` (String) A human-readable description of the object.
- `overrides` (List of Object) List of per-target overrides. Each override replaces the default content for its target. Each object has the following attributes:
  - `target_id` (String, Required) The ID of the target (device, service, or shared policy).
  - `protocol` (String, Required) The override protocol for this target.
  - `value` (String, Optional) The override port value for this target.
- `value` (String) The port value of the service object (e.g., `80`, `8000-8080`). Required for TCP/UDP protocols.

### Read-Only

- `id` (String) The unique identifier of the object. This is a UUID generated by the object service.
